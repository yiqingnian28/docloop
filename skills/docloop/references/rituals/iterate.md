# 仪式 · 开迭代与结算

docloop 以迭代（几天~一周）为心跳。

## 开迭代

前置：now/ 是空的（上迭代已结算）。按 templates/iteration.md 写 `now/iteration.md`：目标、范围（覆盖哪些账本条目 / 模块）、任务清单（T-### 全局递增）、Open Questions。

## 结算清单（按序全过才算关闭）

1. **truth 回写定稿**：设计变化全部改写进 truth，无欠账 Reverse Sync。
2. **账本对账**：迭代范围内每条条目有归属——任务 + 验收证据 / 显式挂起 / 显式否决，**没有一条默默消失**。
3. **inbox 清仓**：每份素材三选一——已出 CH / 显式作废 / 结转新迭代 inbox。
4. **决策增补**：本迭代拍板的长期决策进 `truth/decisions.md`。
5. **写 summary**（模板 templates/summary.md，≤120 行）：做成了什么、关键决策、CH 清单、未了事项（含体检黄项）——它是本迭代唯一可被日常引用的史料。
6. **冻结**：now/ 搬进 `past/YYYY-MM_i##_<标题>/`（任务包连 checkpoint、CH、inbox 一起封存）。**未完任务包例外**：移动结转进新迭代的 now/（checkpoint 随身走，不留冻结副本），summary 记一行"T-### 未完结转 i##"。
7. **开新迭代**：从干净的 now/ 开始（按 templates/iteration.md 写好新 iteration.md）。
8. **体检无红**：红（违反不变量 / 超硬预算）阻断关闭；黄允许关闭但必须逐条写进 summary 未了事项（关闭前允许修订本次 summary）——体检过了，本次结算才算关闭。

## 强制提前结算

没到日历终点但 now/ 总量超预算（默认 2000 行，inbox 只计件数）→ **必须提前结算**：已完成部分按清单结算，未完任务按第 6 步结转。这是防"迭代拖长、文档撑爆上下文"的硬闸门——预算触发，不只靠日历。

## 沉淀三分法

设计定稿 → truth（滚动改写）；重大决策 → decisions.md；过程与出处 → past/ 冻结包 + summary（考古用）。
