# 三层模型细则

> 上级：[总述](README.md)。本章讲每一层的内容物、读写规则，以及模块目录的分层规则。

## truth/ 真相层

**放什么**：架构总纲、已定决策、术语、上游接入（sources/）、各模块技术设计（modules/）、代码地形索引（codemap/）。

**读写规则**：

- 每次变化是**改写成新真相**，不是追加记录。diff 归 git，流水归变更单（CH）。
- 文档内**禁止**出现"进展时间线 / 变更历史"小节——那是旧机制臃肿的根源。
- 每份 truth 文档头部带 YAML frontmatter：`verified: YYYY-MM-DD`（上次人工确认与代码/现实一致的日期，供腐烂检测用）；模块文档可加 `code:`（glob 列表，声明本模块对应的代码路径，子文件未声明时继承父 README——见 [budget-lint.md](budget-lint.md)）。
- 技术设计文档**直接以 `modules/<模块>` 为载体在仓库里写**（人 + AI 双受众），评审走 PR diff。

**模块目录分层规则**（层级由体积预算逼出，不靠事先规划）：

1. 起步单文件：`modules/stock.md`；
2. 超过体积上限（默认 300 行）→ lint 报错，**强制拆成目录**：`modules/sales/README.md`（父真相 = 总述 + 导航，自身也受预算约束）+ 子文件；
3. 子文件再超限 → 递归下钻，规则相同；
4. **最深 3 层**（以 modules/ 为根计路径深度，`modules/a/b/c.md` 即第 3 层封顶）——再深说明模块边界该重切了，体检提示而不是继续加深。

目录树因此永远和实际复杂度成正比，不会出现为规范空转的深目录。

**codemap/ 放真相层的理由**：它是"代码现状的索引"，也是最易腐烂的文档（代码一动就过时），正好吃 `verified:` + 腐烂检测的治理。

## now/ 当下层

**放什么**：本迭代的一切活文档——

| 文件 | 用途 |
|---|---|
| `iteration.md` | 迭代目标 / 范围 / 任务清单。**项目状态唯一的家**（规矩文件不放状态） |
| `tasks/T-###_<名>.md` | 任务包 = sdd-riper Feature Spec（目标 / In-Out / Done Contract / checkpoint / 验证 / 偏差） |
| `changes/CH-###_<名>.md` | 变更单（见 [change.md](change.md)） |
| `inbox/` | 原始素材收件箱：纪要 / 聊天 / 口头需求原样存，待裁决 |
| `context/`、`handoff/` | sdd-riper 的临时产物 |

**读写规则**：

- now/ 只活一个迭代；结算时**整体原样冻结**搬进 past/（未完任务包例外：移动结转进新迭代，见 [lifecycle.md](lifecycle.md)），新迭代从空的 now/ 开始。
- inbox 内容**不进模型上下文**——它只是 CH 单的附件线索。
- now/ 总量有预算；超限触发**强制提前结算**（见 [budget-lint.md](budget-lint.md)）。

## past/ 史料层

**放什么**：每迭代一个目录 `YYYY-MM_i##_<标题>/`，内含：

- `summary.md`（≤1 页）：迭代摘要——做成了什么、关键决策、CH 清单、未了事项。**唯一可被日常引用的史料。**
- now/ 的原样冻结（任务包连 checkpoint 一起封存）。

**读写规则**：

- 只增不改；**默认永不进模型上下文**。
- 考古协议：先读 summary → 需要细节再按链接精读单份冻结文档，禁止全量回读。
- "怎么一步步来的"由四层留痕覆盖：git 历史（真相 diff）+ CH 单（传导链）+ 迭代冻结包（过程原件）+ summary（导航）。因此**不设逐节点冻结快照仪式**（旧机制 iterations/ 退役，决策 D-005）。
