# docloop 机制规范 · 总述

> 本目录是机制的**当前真相**（滚动改写·历史看 git）。一句话：**按知识的时间性分三层，变更单当一等公民，体积预算 + 定期体检强制文档不长胖、不腐烂。**

## 解决什么问题

1. **变更臃肿**：需求频繁变更 → 设计文档变 → 代码变，变化记录逐层堆积，文档随时间无限长胖，难以维护。
2. **人是不可靠信源**：PRD / 设计稿 / 会议纪要体积不受控、互相矛盾、会出错，直接喂 AI 既塞爆上下文又引入错误。
3. **上下文超载**：迭代拉长后文档总量超出模型上下文的驾驭范围——上下文有限且宝贵。
4. **无人校准**：目录长歪、文档和代码脱节（腐烂）没人发现。

## 核心模型：按时间性分三层

旧做法按**内容**分类（设计一堆、执行一堆），历史和现状混在同一批文件里长胖。docloop 改成按**知识的年龄**分：

| 层 | 性质 | 增长规律 |
|---|---|---|
| `truth/` 真相层 | 当前设计真相，滚动**改写** | 只随系统复杂度增长 |
| `now/` 当下层 | 本迭代活文档（任务包 / 变更单 / 素材收件箱） | 迭代内增长，结束**清空** |
| `past/` 史料层 | 迭代冻结包，只增不改 | 随时间增长，但**默认不进上下文** |

## 目标项目目录全貌

```
docs/
├── README.md                 # 导航 + 开工协议
├── truth/                    # 真相层（滚动改写·体积受限）
│   ├── architecture.md       #   架构总纲
│   ├── decisions.md          #   已定决策（ADR 索引）
│   ├── glossary.md           #   术语
│   ├── sources/              #   上游接入（人产文档 → AI 可消化）
│   │   ├── index.md          #     登记簿：每份上游文档一行
│   │   └── S-##_<名>/
│   │       ├── digest.md     #     摘编（对齐契约·≤150 行）
│   │       └── items.md      #     条目账本（逐条对账·可拆章节）
│   ├── modules/<模块>[/...]  #   模块技术设计：单文件起步·超预算强制分层·≤3 层
│   └── codemap/              #   代码地形索引（易腐·吃腐烂检测）
├── now/                      # 当下层（本迭代·结束清空）
│   ├── iteration.md          #   本迭代目标 / 范围 / 任务清单（项目状态唯一的家）
│   ├── tasks/T-###_<名>.md   #   任务包（= sdd-riper Feature Spec）
│   ├── changes/CH-###_<名>.md#   变更单
│   ├── inbox/                #   原始素材收件箱（纪要/聊天·待裁决·不进上下文）
│   ├── context/              #   sdd-riper context bundle（临时）
│   └── handoff/              #   交接包（临时）
└── past/                     # 史料层（只增不改·默认不进上下文）
    └── YYYY-MM_i##_<标题>/
        ├── summary.md        #   迭代摘要（≤1 页·唯一可被日常引用的史料）
        └── …                 #   now/ 原样冻结搬入
```

## 五条不变量（机制宪法）

1. **truth 只随系统复杂度增长，不随时间增长**——每次变化是"改写"不是"追加"，truth 内不留任何时间线 / 变更流水（历史归 git 和 past/）。
2. **now 每迭代结束必清空**——整体冻结搬进 past/，新迭代从干净的 now/ 开始。
3. **past 默认不进模型上下文**——唯一可被日常引用的史料是各迭代 `summary.md`（≤1 页）。
4. **规矩文件只放规矩，不放项目状态**——AGENTS.md / CLAUDE.md 里没有"现在到哪了"；状态活在 `now/iteration.md`。
5. **权威层只能经 CH 裁决改写**——原始素材（纪要 / 聊天 / 口头需求）永不直接成为实现依据。
