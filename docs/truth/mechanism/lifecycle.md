# 迭代生命周期：四个仪式

> 上级：[总述](README.md)。docloop 以**迭代**（几天~一周）为心跳；文档的流动由四个仪式驱动。

| 仪式 | 时机 | 干什么 |
|---|---|---|
| **开迭代** | 迭代起点 | 写 `now/iteration.md`（目标 / 范围 / 任务清单） |
| **记变更** | 需求一变就记 | inbox → CH 裁决 → 改写权威层（见 [change.md](change.md)） |
| **体检** | 结算时必跑 + 随时可跑 | 目录合规 / 体积超限 / 死链 / 腐烂 / 孤儿条目 / inbox 积压（见 [budget-lint.md](budget-lint.md)） |
| **结算** | 迭代终点 | 见下方清单 |

## 开迭代

- `now/` 必须是空的（上迭代已结算）；
- `iteration.md` 写清：迭代目标、范围（本迭代覆盖哪些账本条目 / 模块）、任务清单（T-###）、Open Questions。

## 结算清单（按序执行，全过才算关闭）

1. **truth 回写定稿**：本迭代所有设计变化已改写进 truth（无欠账的 Reverse Sync）；
2. **账本对账**：迭代范围内每条账本条目有归属——任务 + 验收证据 / 显式挂起 / 显式否决，**没有一条默默消失**；
3. **inbox 清仓**：每份素材三选一——已出 CH / 显式作废 / 结转新迭代 inbox；未裁决素材禁止默默冻结；
4. **决策增补**：本迭代拍板的长期决策进 `truth/decisions.md`；
5. **写 summary**（≤1 页）：做成了什么、关键决策、CH 清单、未了事项（含体检黄项）——它是这个迭代**唯一可被日常引用的史料**；
6. **冻结**：now/ 搬进 `past/YYYY-MM_i##_<标题>/`（任务包连 checkpoint、CH、inbox 一起封存）。**未完任务包例外**：移动结转进新迭代的 now/（checkpoint 随身走，不留冻结副本），summary 记一行"T-### 未完结转 i##"；
7. **体检无红**：红（违反不变量 / 超硬预算）阻断关闭；黄（腐烂嫌疑 / 积压）允许关闭，但必须逐条写进 summary 未了事项；
8. **开新迭代**：从干净的 now/ 开始。

## 强制提前结算（预算触发，不只靠日历）

迭代还没到期，但 now/ 总量超预算 → **必须提前结算**：把已完成部分按上述清单结算，未完任务按清单第 6 步移动结转，作为新迭代的开局任务。这就是"避免迭代周期过长、文档体积超出模型驾驭范围"的硬闸门。

## 迭代级沉淀去哪了

迭代做完，"当时的设计稿、重大决策"三处各司其职：

- 设计定稿 → **truth**（当前真相，滚动改写）；
- 重大决策 → **decisions.md**（长期有效的拍板）；
- 过程与出处 → **past/ 冻结包 + summary**（考古用）。
