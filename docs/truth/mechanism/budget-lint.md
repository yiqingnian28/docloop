# 上下文预算 + 体检（lint）规则

> 上级：[总述](README.md)。核心立场：**"模型驾驭得住"不是愿望，是可校验的硬指标。** 上下文有限且宝贵。

## 开工必读集（boot set）

新会话开工，AI 必读的集合定义为：

```
规矩文件（AGENTS.md/CLAUDE.md）
+ docs/README.md（导航）
+ now/iteration.md
+ 手头任务包
+ 相关模块 truth
+ 相关摘编（digest）
```

体检脚本**实际计算**这个集合的总行数，超预算 = lint 失败，必须拆分或压缩。

## 体积预算（默认值·做成配置可调·待实战校准）

| 对象 | 上限 |
|---|---|
| 单模块 truth 文档 | 300 行（超限强制拆目录，见 [layers.md](layers.md)） |
| 任务包 T-### | 250 行 |
| 变更单 CH-### | 80 行 |
| 摘编 digest.md | 150 行 |
| 迭代摘要 summary.md | 120 行（≈1 页） |
| **开工必读集合计** | **2000 行** |
| now/ 总量 | 超限 → 强制提前结算（见 [lifecycle.md](lifecycle.md)） |

## 渐进披露协议（读的纪律）

- 默认读摘编 / 真相，不读上游全文；
- 需要细节时按锚点深读**原文特定章节**（实现时按账本条目拉原文，见 [upstream.md](upstream.md)）；
- past/ 默认不读，考古先读 summary 再按链接精读单份；
- inbox 不进上下文。

## 体检（lint）六项

| # | 检查 | 判定 |
|---|---|---|
| 1 | **目录合规** | 目录结构 / 文件命名符合规范（truth/now/past 骨架、`T-###`/`CH-###`/`S-##` 命名、模块 ≤3 层） |
| 2 | **体积超限** | 上表各预算 + boot set 实算 |
| 3 | **死链** | 文档间链接、CH→条目→任务的引用是否指向存在的目标 |
| 4 | **腐烂检测** | truth 文档头 `verified: <日期>` vs 该模块代码的 git 提交活跃度——代码动了很多但 verified 很旧 → 标黄"疑似腐烂" |
| 5 | **孤儿条目** | 账本里长期无归属的条目；任务声称完成但对应条目未勾 |
| 6 | **inbox 积压** | 未裁决素材躺尸 → 标黄提醒 |

结果分级：**红**（违反不变量 / 超硬预算，结算被阻断）· **黄**（疑似腐烂 / 积压，提醒不阻断）。
