# 上下文预算 + 体检（lint）规则

> 上级：[总述](README.md)。核心立场：**"模型驾驭得住"不是愿望，是可校验的硬指标。** 上下文有限且宝贵。

## 开工必读集（boot set）

新会话开工，AI 必读的集合定义为：

```
规矩文件（AGENTS.md/CLAUDE.md）
+ 已装 skill 常驻内核（docloop 的 SKILL.md 等）
+ docs/README.md（导航）
+ now/iteration.md
+ 手头任务包
+ 任务包 frontmatter 声明的必读集（相关模块 truth + 相关摘编）
```

任务包用 frontmatter 显式声明自己的必读集；体检脚本对**每个开放任务**实算一遍 boot set（行数 + 字节双口径），取最大值判超限——超预算 = lint 失败，必须拆分或压缩。

## 体积预算（默认值·做成配置可调·待实战校准）

| 对象 | 上限 |
|---|---|
| 单模块 truth 文档 | 300 行（超限强制拆目录，见 [layers.md](layers.md)） |
| 任务包 T-### | 250 行 |
| 变更单 CH-### | 80 行 |
| 摘编 digest.md | 150 行 |
| 迭代摘要 summary.md | 120 行（≈1 页） |
| SKILL.md 常驻内核 | 150 行 |
| **开工必读集合计** | **2000 行** |
| now/ 总量 | 2000 行（inbox 只计件数不计行数）；超限 → 强制提前结算（见 [lifecycle.md](lifecycle.md)） |

- 未单列的 truth 文档（items.md、sources/index.md 等）一律按 300 行档；items 超限按章节拆目录。
- 每档行数预算配套字节上限 = 行数 × 65（字节 ÷ 3.5 ≈ token）；lint 双指标，任一超即超——防"行数合规的超长行"。

## 渐进披露协议（读的纪律）

- 默认读摘编 / 真相，不读上游全文；
- 需要细节时按锚点深读**原文特定章节**（实现时按账本条目拉原文，见 [upstream.md](upstream.md)）；
- past/ 默认不读，考古先读 summary 再按链接精读单份；
- inbox 不进上下文。

## 体检（lint）六项

| # | 检查 | 判定 |
|---|---|---|
| 1 | **目录合规** | 目录结构 / 文件命名符合规范（truth/now/past 骨架、`T-###`/`CH-###`/`S-##` 命名、模块 ≤3 层） |
| 2 | **体积超限** | 上表各预算 + boot set 实算 |
| 3 | **死链** | 文档间链接是否指向存在的目标；CH/T/S 的 ID 引用在 now/ 与 past/ 两处解析 |
| 4 | **腐烂检测** | truth 文档 frontmatter `verified:` vs `code:` glob 内代码的 git 提交活跃度——代码动了很多但 verified 很旧 → 标黄"疑似腐烂"；未声明 `code:` 的文档退化为纯日期阈值（过久未核标黄） |
| 5 | **孤儿条目** | 账本里未排期且跨过 ≥1 个已结算迭代的条目（v1 近似：past 非空即对未排期条目提醒）；任务包"覆盖条目"声称完成但账本未勾 |
| 6 | **inbox 积压** | 未裁决素材躺尸 → 标黄提醒 |

结果分级：**红**（违反不变量 / 超硬预算，结算被阻断）· **黄**（疑似腐烂 / 积压，提醒不阻断）。

## 实现形态

零依赖 Node（≥18）单脚本，随 skill 发行：`skills/docloop/scripts/docloop_lint.mjs`，目标项目里一条命令跑完；预算默认值内置，可被目标项目根的 `docloop.config.json` 覆盖（含模块根等路径配置）。结算仪式必跑，平时随手可跑。（选型理由见 decisions.md D-012。）
