# 支持用例全景（验收范围）

> 上级：[总述](README.md)。7 类 40 例；机制任何改动以不破坏本表为底线，模板与 lint 按本表验收。

## A. 知识组织 — "东西住在哪"

| # | 用例 | 机制承接 |
|---|---|---|
| UC-A1 | 查某模块当前设计真相（数据模型/接口/状态机） | `truth/modules/<模块>` 滚动改写，看它=最新 |
| UC-A2 | 查架构总纲 / 技术决策 / 术语 | `truth/architecture.md` / `decisions.md` / `glossary.md` |
| UC-A3 | 模块变复杂后拆分子模块 | 体积超预算 → lint 强制拆目录，递归分层（≤3 层） |
| UC-A4 | 进陌生代码前查代码地形 | `truth/codemap/`（sdd-riper 产物·吃腐烂检测） |
| UC-A5 | 同一事实只在一处权威 | 单一真相源不变量：别处只链接不复述 |
| UC-A6 | 新项目装机：初始化 docs/ 三层骨架 + AGENTS.md 落点映射 | skill 装机仪式（init reference + 全套模板） |

## B. 上游接入 — "人产的文档怎么进来"

| # | 用例 | 机制承接 |
|---|---|---|
| UC-B1 | PM 定稿一份 PRD，让 AI 能消化 | 挂锚（登记簿）+ 摘编 digest（≤150 行对齐契约·人评审确认） |
| UC-B2 | 确保产品细节全部被实现，不默默丢失 | 条目账本 items：逐条枚举，四状态（未排期/进行中/已实现+证据/明确不做） |
| UC-B3 | 0→1 大 PRD 不必一次拆完 | 账本渐进细化：先章节级粗条目，临开工再细化，任何时点覆盖完整 |
| UC-B4 | 设计稿（Figma 等仓库外产物）接入 | 挂锚 + 摘编（交互规则/状态清单 + 关键帧链接） |
| UC-B5 | 技术设计文档的产出与评审 | 直接以 `truth/modules/` 为载体在仓库写，评审走 PR diff |
| UC-B6 | 会议纪要/聊天记录/口头需求先收起来 | `now/inbox/` 原样存，永不直接作为实现依据 |

## C. 变更管理 — "需求变了怎么办"

| # | 用例 | 机制承接 |
|---|---|---|
| UC-C1 | 一次需求变化的传导链可追（变了啥→波及哪些设计→哪些任务） | CH 变更单：变化/波及/处置三段，链接集中一张纸 |
| UC-C2 | 新说法与已有需求/历史裁决矛盾时不混入 | AI 起草 CH 时强制产出冲突检查报告（对照账本 + truth 含 decisions + 本迭代 CH） |
| UC-C3 | 矛盾由人拍板，有名有姓可翻案 | CH 单记录裁决人+结论；裁决后才允许改写权威层 |
| UC-C4 | 变更不让真相文档长胖 | truth 只被改写（diff 归 git），流水归 CH 单，CH 随迭代冻结 |
| UC-C5 | 变更暂不做也不丢 | CH 处置=挂起待排，体检盯着 |
| UC-C6 | "开会说了但没人落单"不漏 | inbox 未裁决素材 → 体检标黄提醒 |

## D. 迭代生命周期 — "以迭代为心跳"

| # | 用例 | 机制承接 |
|---|---|---|
| UC-D1 | 开迭代：定目标/范围/任务清单 | `now/iteration.md` |
| UC-D2 | 迭代内任务怎么一步步推进的留痕 | `now/tasks/T-###`（=Feature Spec，内嵌 RIPER checkpoint） |
| UC-D3 | 结算：定稿→摘要→冻结→开新 | 结算清单（见 lifecycle.md） |
| UC-D4 | 结算时需求不漏项对账 | 账本对账：每条目必有归属，禁止默默消失 |
| UC-D5 | 迭代拖太长文档撑爆之前止损 | now/ 超预算 → 强制提前结算（预算触发，不只靠日历） |
| UC-D6 | 迭代级设计稿/重大决策沉淀 | truth 定稿 + decisions 增补 + past/ 冻结包，三处各司其职 |

## E. 任务执行 — "单个任务怎么干"（sdd-riper 分工）

| # | 用例 | 机制承接 |
|---|---|---|
| UC-E1 | 任务的目标/边界/Done Contract/验证/门禁 | sdd-riper（one/light）驱动，产物落 `now/tasks/` |
| UC-E2 | 实现时细节不失真 | 任务包按账本条目拉原文章节完整细节（摘编只是导航） |
| UC-E3 | 换对话/长暂停/交接恢复 | handoff 包落 `now/handoff/`，随迭代冻结 |
| UC-E4 | 实现中发现设计要改 | Reverse Sync：回改 truth（+必要时补 CH），任务包只记偏差 |

## F. 质量校准 — "文档腐没腐、目录歪没歪"

| # | 用例 | 检查项 |
|---|---|---|
| UC-F1 | 目录合规：文件长没长歪、命名对不对 | 目录/命名规范校验 |
| UC-F2 | 体积超限：单文档 & now/ 总量 & 开工必读集 | 各级预算校验（可配置） |
| UC-F3 | 死链：链接的文件/条目还在不在 | 链接校验 |
| UC-F4 | 腐烂检测：代码动了很多、文档很久没核 | `verified:` 日期 vs git 活跃度比对，标黄 |
| UC-F5 | 孤儿条目：账本里长期无归属的需求 | 账本状态扫描 |
| UC-F6 | 未裁决积压：inbox 躺尸素材 | inbox 扫描标黄 |

## G. 上下文经济 + 考古 — "AI 驾驭得住 & 历史查得到"

| # | 用例 | 机制承接 |
|---|---|---|
| UC-G1 | 新会话开工必读集有界 | boot set 定义 + 计入预算硬校验 |
| UC-G2 | 渐进披露：默认读摘编，需要时按锚深读原文特定章节 | 阅读协议；上游全文/inbox/past 默认不进上下文 |
| UC-G3 | 考古"这个决策怎么来的" | past/ 冻结包 + CH 单 + decisions.md + git 历史 |
| UC-G4 | 考古"这条需求最后做了没、证据呢" | 账本条目状态 + 验收证据链接 |
| UC-G5 | 日常引用历史但不背全量史料 | 唯一可引用史料 = `past/<迭代>/summary.md`（≤1 页） |
| UC-G6 | 多工具通用（Claude Code / Codex / Cursor） | 知识层纯 Markdown+约定；AGENTS.md 声明落点映射 |

## 暂不纳入 v1 的边缘用例

- **多人并行**：多条任务线/多迭代并行时 now/ 怎么组织（v1 假设单迭代串行）——OQ-1；
- **跨仓库**：一个产品多代码仓时 truth 放哪（v1 假设单仓）——OQ-2。
