## 变化

GitHub issue #2(yiqingnian28,2026-07-07,Related #1)反馈实战迁移中的新问题:正文没有"变更历史"标题、历史词也不密,但把当前规则、裁决 ID、日期、机器预检证据、代码锚点、排期估算混排在同一段/同一行,阅读面仍像审计报告——第 7 项(CH-003)抓不到这类"证据/裁决噪声"。提炼四个变化点:

1. **truth-like 扫描路径**:新增配置 `truthDirs`(默认 `['docs/truth']`,可追加迁移期路径),仅作用于阅读面检查(第 7 项),结构类检查(1–6)照旧只认装机结构;docs/ 存在性门槛放宽(docs/ 或 truthDirs 至少一个存在,未装机时只跑阅读面)。**docs/truth 之外的路径一律只黄不红**——不变量约束只对已装机结构有强制力。定位为迁移期过渡配置,文档明示装机是正道。
2. **阅读面噪声检查**(第 7 项扩展,全部只出黄):7b 噪声行——散文/bullet 行(表格行、标题不计)命中 ≥3 类信号(编号、日期、裸代码锚点、排期/过程词、超长括号)计噪声,占比超阈标黄;7c 超长单元——bullet/表格单元格超字符阈值标黄;台账类密集是契约,只受 7c 约束。规范引用(行内代码、markdown 链接)不计信号。词表补安全过程词(`机器预检`/`附C`/`待落码`/`人天`;"证据"这类常用词不进默认表)。
3. **rewrite recipe**:按"agent 可直接执行的提示词"标准写进修复指引——逐句分类(规则/边界/证据/追溯/排期)→ 按去向搬家(规则表、边界表、证据移任务包、裁决归 CH、状态归 iteration)→ 复跑 lint 验证。
4. **提示语升级**:噪声黄项输出附示例行 + recipe 指引,不只报密度数字。

**架构红线**(沉淀为 D-014):探测归零依赖脚本(确定性、可进 CI),语义判定与改写归 skill 内 agent(recipe 即提示词),拍板归人;结算门禁必须确定性,LLM 永不进阻断面。
