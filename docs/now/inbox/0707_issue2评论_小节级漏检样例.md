## 漏检现象

`docloop_lint` 对整份 `细节-03-主页聚合与前端接口.md` 没有报黄，但人工检查发现两个小节明显仍是“当前规则 + 历史反转 + 证据锚点 + 欠账状态”混写：

1. `【细3.9】icon 栏锚定三要素(裁定反转:代码含 JUMP)`
   - 标题含“裁定反转”，已经是历史/追溯口吻。
   - 正文有“旧文档打架 / 旧文档反而对 / 反转已核验 / 附C”等证据与考古信息。
   - `anchorType` 当前规则、代码证据、旧口径差异、`JUMP` 待确认揉在一个 bullet 里。

2. `【细3.11】累计节省 / 动效字段欠账`
   - 标题含“字段欠账”，小标题含“多次反转”。
   - “反转脉络”“机器预检✓/附C/Result:105/108”“原 VO 未落核验结论已过时”等应归追溯/证据。
   - 当前字段口径、数据源欠账、排期状态、PRD 矛盾混在同一阅读面。

这些段落后来按 recipe 改成了：当前规则表 + 欠账/待确认表 + 追溯/证据链接，阅读体验明显变好。

## 为什么现有 lint 没发现

按当前脚本逻辑看，原因主要是阈值和规则粒度偏保守：

- 现在主要按“整文件”计算噪声密度，局部小节很脏但全文件比例没超阈值时不会报。
- 标题检测只抓“变更历史 / 修订记录 / 进展时间线 / changelog”等强模式，没有抓“裁定反转 / 口径反转 / 多次反转 / 字段欠账 / 编码现状 / 已落码 / 核验结论已过时”等 truth-like 阅读面常见噪声标题。
- 超长单元阈值是 200 字，漏检段落多在 185~196 字之间，刚好低于阈值；其中一条原始 209 字 bullet 因剥掉行内代码后降到 189，也不会报。
- 7b 要求单行命中至少 3 类信号，且按文件级比例报；“少量但非常典型”的噪声行会被稀释。
- `欠账 / 未定 / 待裁决 / 反转 / 已落码 / 内部矛盾 / 机器预检 / 附C / 核验结论` 这类词需要区分：单独出现不一定错，但出现在标题或同一 bullet 中与日期、编号、代码锚点共现时，应强提示拆分。

## 建议优化

1. 增加“小节级”噪声扫描：按 heading section 计算历史词/噪声行密度，命中后报告小节标题和示例行。
2. 扩展 truth-like 噪声标题词表：`裁定反转`、`口径反转`、`多次反转`、`字段欠账`、`编码现状`、`机器预检`、`已落码`、`核验结论`。
3. 增加“近阈值长 bullet”提示：例如 180~200 字且含 `#\d+`、日期、`附C`、代码锚点、`待确认/待裁决/欠账` 时也报黄。
4. 对标题 + 首段 blockquote 做特殊检查：如果出现“为什么要展开 + 旧文档/反转/核验/附C/排期/欠账”等组合，提示改为“当前读者/权威来源/追溯链接”。
5. 输出建议保持现有 recipe：当前规则表、实现边界表、欠账/待确认表、追溯/证据链接。

这个问题和 issue 本体一致：不是单纯长短问题，而是“主阅读面被局部证据账/考古账污染”。
