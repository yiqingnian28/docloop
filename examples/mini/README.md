# examples/mini · 书签管家（docloop 完整生命周期走读）

一个虚构的命令行书签工具，演示 docloop 机制从装机到第二个迭代的全过程。**读完这一个目录，等于把机制的主干用例都过了一遍。**

|---|---|---|
| 06-20 | 装机 + 开 i01；需求会上提出加标签 | 骨架 + `past/.../inbox/0620需求会纪要.md`（原样存） |
| 06-25 | CH-001 裁决：PRD 升 v1.1，标签排下迭代 | `past/.../changes/CH-001` → 登记簿升版、账本加 R-003 |

| 07-01 | 开 i02：标签搜索 | `now/iteration.md` + `now/tasks/T-003` |
| 07-04 | 老板群里要导入导出 → CH-002 裁决：挂起待排 | `now/inbox/0704聊天记录.md` → `now/changes/CH-002` → 账本加 R-004 |
| 07-05 | 晨会又冒出"按时间排序"，还没人落单 | `now/inbox/0705晨会纪要.md`（体检会标黄） |

## 机制要点在哪看

- **锚 + 摘编 + 账本**（永不复制全文）：`docs/truth/sources/`——登记簿一行、digest ≤150 行、items 一行一条四状态
- **变更两种裁决**：升版照改（CH-001）vs 挂起待排（CH-002，条目回链、体检盯着）
- **truth 滚动改写**：`truth/modules/bookmark.md` 只有当前真相，标签行为直接写进去，没有"变更历史"小节
- **结算冻结**：`past/2026-06_i01_骨架/` 是 i01 的 now 原样封存；日常只引用它的 `summary.md`
- **考古**："标签怎么来的？"→ summary 一行 → CH-001 传导链 → inbox 原话，三跳到底

## 跑体检

在 docloop 仓库根执行：

```
node skills/docloop/scripts/docloop_lint.mjs examples/mini
```

预期 **0 红 2 黄**——两处是故意留的教学示例：

1. `S-01.R-004 未排期`：挂起的需求跨迭代躺着，体检提醒别忘了它（CH-002 的处置结果）；
2. `inbox/0705晨会纪要.md 未裁决`："开会说了但没人落单"的经典漏项，体检兜底。

结算 i02 时这两条黄必须处置或写进 summary 未了事项，否则不算干净关闭。
